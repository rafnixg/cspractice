# Instrucciones de desarrollo para PracCentral

## Configuración del entorno

1. Clonar el repositorio y restaurar dependencias:
   ```bash
   git clone <repo>
   cd cspractice
   dotnet restore PracCentral.sln
   ```

2. Build del proyecto:
   ```bash
   dotnet build -c Debug
   ```

3. Ejecutar pruebas:
   ```bash
   dotnet test PracCentral.sln
   ```

## Estructura de carpetas

- `src/PracCentral/` — Código principal del plugin
- `tests/PracCentral.Tests/` — Suite de pruebas
- `data/prefire/` — Configuraciones de mapas para Prefire

## Agregar un nuevo módulo

1. Crear carpeta: `src/PracCentral/Modules/MyModule/`
2. Implementar `IPracModule`:
   ```csharp
   public sealed class MyModule : IPracModule
   {
       public string Name => "MyModule";
       
       public void Load(ModuleContext context) { }
       public void Unload(ModuleContext context) { }
       public void Dispose() { }
   }
   ```
3. Registrarlo en `Main.cs` bajo `_moduleByMode`
4. Agregar tests en `tests/PracCentral.Tests/`

## Guía de estilo

- **Nombres**: PascalCase para clases, camelCase para variables
- **Métodos**: Documentar parámetros y comportamiento esperado
- **Logging**: Usar `context.Logger` (nunca `Console.WriteLine` directo)
- **Async**: Todo I/O de disco debe ser async; sincronizar con `MainThreadDispatcher.Enqueue()`
- **Disposables**: Siempre registrar en `EventSubscriptionRegistry` cuando sea aplicable

## Pruebas

Cada módulo debe tener:
- Tests de load/unload
- Tests de estado y transiciones
- Tests de validación de input (si aplica)

Ejecutar con coverage:
```bash
dotnet test PracCentral.sln /p:CollectCoverage=true
```

## Debugging

1. Abrir el proyecto en VS Code:
   ```bash
   code .
   ```

2. Configurar breakpoints y ejecutar tests desde la paleta de comandos.

3. Para debuggear en servidor vivo:
   - Adjuntar el debugger de .NET al proceso `dotnet` del servidor CS2

## Notas de implementación

- **No crear números mágicos**: Todos los valores deben ser constantes tipadas en `Config/`.
- **Usar records para DTOs**: Aprovechar record types para modelos JSON.
- **ConVarSnapshotService**: Siempre capturar antes de modificar, restaurar al salir.
- **Memory Leaks**: Verificar con `EventSubscriptionRegistry` que no queden listeners huérfanos.
